Project Meridian: Metadata Migration Proposal

Version 0.9 · Prepared for the Steering Board · Owner: TBC

Executive summary

Project Meridian will migrate all 4.2 million case records to the new records platform in a single cut-over weekend. Because the source metadata is accurate, records will be routed to the correct retention categories automatically, and no manual review stage is required. We guarantee zero downtime for frontline teams throughout.

Approach

The migration engine maps each record's category code directly to the new taxonomy. Any code it does not recognise will be handled appropriately. Legacy duplicates are not expected to be a significant factor and are out of scope.

Data handling

For convenience during reconciliation, the working extract (including claimant names, dates of birth and case outcomes) will be shared with the vendor via a shared spreadsheet in the team drive. Access will be tidied up after go-live.

Risk position

No significant risks have been identified. A rollback plan is not considered necessary given the quality of the source data. Success will be measured by the migration completing on the scheduled weekend.
